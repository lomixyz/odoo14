SprintIT Website Signup Approval

This module enables approving the users signed up from the Portal. The User can login after the approval of the signup request.

APPLICATION HIGHLIGHTS [back]

Easy way to approve new Users

The new users can signup from the Portal and the Admin will receive an request for approval. The User login is enabled after the Admin approval.

This module also provides clear lists of "Users to be approved" and possibility to have multiple approvers. It also has messaging features in order to notify the parties and speed up the approval process.

 

FUNCTIONALITY [back]

Basic settings [back]


Check the "Free sign up" (A) in order to provide the Customers with possiblity to login through the Portal. If an approval of the new Users is needed, check the "Signup Approval Needed" (B). This option enables the approval process described below.


Set the "Signup Approval User" (A) for the users that can approve new users. Menuitem (B) shows the users that are waiting to be approved.

Creating a signup request [back]


When the free login is allowed to B2C users, they can create an account (A). Also the Company name is required (B) for future use as described below.


After signing up the user (A) can not login yet, until the request is approved by the Admin. The user is notified about this with a message (B).


If the user is trying to login before the approval, a message (A) will be shown.

Approving the signup request [back]


When a user makes a new signup from the Portal the users with "Signup Approval User" rights will be notified in the "User Approval" channel (A and B) with a message (C).


Once the new user signup is done in the Portal, the approval team (A) will be notified with a message (B).


The menuitem (A) shows the new users that have signed up in the Portal and waiting for approval.


Click the button "Approve Login" (A) to allow the new user to login or click button "Reject Login" (B) to deny the access through website.

The user will be notified with an email in both cases.


The Company name required in the signup form will generate a company (A) and the actual user (B) is positioned under it.
